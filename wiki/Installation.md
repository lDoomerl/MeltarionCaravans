# Установка

## Требования

- Paper/Leaf `1.21.x`
- Java `21`
- `Towny`
- `Dynmap` — опционально

### Что зависит от Towny

Towny нужен для:

- проверки спавна каравана в своём городе;
- проверки `Shop Plot` для публичной торговли;
- выбора городов и Shop Plot для маршрутов.

Если Towny отсутствует, плагин в целом может загрузиться, но Towny-зависимые механики будут недоступны.

## Установка jar

1. Скачайте актуальный jar.
2. Поместите его в `plugins/`.
3. Убедитесь, что на сервере используется Java 21.
4. Запустите сервер.

## Первый запуск

При первом запуске плагин:

1. создаёт папку данных;
2. создаёт или обновляет ресурсные файлы;
3. поднимает SQLite-базу `plugins/MeltarionCaravans/caravans.db`;
4. загружает караваны, инвентари, торговые операции и маршруты.

## Где создаются config.yml, lang.yml и gui.yml

Файлы создаются в папке:

```text
plugins/MeltarionCaravans/
```

Там же будут:

- `config.yml`
- `lang.yml`
- `gui.yml`
- `caravans.db`

## Обновление плагина

Обычная схема обновления:

1. Остановите сервер.
2. Замените старый jar новым.
3. Запустите сервер снова.

Удалять конфиги вручную **не нужно**.

## Как работают config-version и автообновление

Во всех трёх ресурсных файлах есть:

```yml
config-version: 1
```

На старте и при `/caravan reload` плагин:

- сравнивает версию файла из папки сервера с версией файла из jar;
- если версия старее, делает backup;
- берёт свежий шаблон из jar;
- переносит в него ваши текущие значения;
- дописывает новые ключи;
- обновляет `config-version`.

### Важные правила

- значения администратора сохраняются;
- новые ключи добавляются автоматически;
- неизвестные кастомные ключи не удаляются;
- если YAML повреждён, создаётся backup и файл пересоздаётся из дефолта.
