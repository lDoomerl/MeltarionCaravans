# Чеклист администратора

## 1. Startup

- Сервер стартует без stacktrace.
- `config.yml`, `lang.yml`, `gui.yml` созданы.
- `caravans.db` создан.
- В логе нет ошибок Towny/Dynmap/SQLite.

## 2. License

- `/caravan admin givelicense <ник> 1` работает.
- Игрок получает предмет лицензии.
- Лицензия распознаётся корректно.

## 3. Creation

- Игрок создаёт караван через ПКМ по лицензии.
- Лимиты по permission работают.
- Имя по умолчанию создаётся корректно.

## 4. Physical Spawn

- `/caravan spawn <id>` работает в своём городе.
- Вне своего города команда отклоняется.
- Проекция появляется как trader + 2 llama.

## 5. Setup GUI

- Правый клик владельца открывает setup GUI.
- `/caravan admin setup <id>` работает.
- Кнопки GUI открывают нужные разделы.

## 6. Trading

- SELL-операция создаётся.
- BUY-операция создаётся.
- Публичная торговля работает только на Shop Plot.
- Публичная торговля блокируется вне Shop Plot.

## 7. Movement

- `/caravan move <id> <x> <z>` запускает движение.
- `/caravan stop <id>` останавливает движение.
- `/caravan return <id>` отправляет домой.
- ETA меняется логично.

## 8. Routes

- Route GUI открывается.
- Добавление точки через выбор города работает.
- Ввод времени стоянки через чат работает.
- `Start Route` запускает маршрут.
- `Loop Route` переключается.

## 9. Dynmap

- При наличии Dynmap появляются маркеры.
- Подпись маркера обновляется.
- При отсутствии Dynmap плагин не падает.

## 10. Attack Behavior

- Урон по сущности проекции уменьшает HP каравана.
- Статус становится `ATTACKED`.
- Если включено, движение ставится на паузу.
- Если включено, auto-resume срабатывает после задержки.

## 11. Reload Behavior

- `/caravan reload` проходит без ошибок.
- `/caravan admin reload` проходит без ошибок.
- Конфиги и ресурсы обновляются безопасно.
- Активные караваны не теряются.
