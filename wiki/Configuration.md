# Конфигурация

## Структура файлов

### config.yml

Содержит:

- игровые настройки;
- системные настройки;
- движение;
- маршруты;
- интеграции;
- лицензию;
- отладку.

### lang.yml

Содержит:

- сообщения игрокам;
- сообщения администраторам;
- help-текст;
- debug/info строки;
- уведомления.

### gui.yml

Содержит:

- заголовки GUI;
- названия кнопок;
- lore;
- материалы иконок.

## Разделы config.yml

### currency-item

Предмет-валюта для торговли.

Пример:

```yml
currency-item: GOLD_INGOT
```

### caravan

Настройки самого каравана:

- `max-name-length`
- `inventory-size`
- `health`
- `damage-multiplier`
- `visual-health-sync-interval-ticks`

### caravan-license

Настройки предмета лицензии:

- включение системы лицензий;
- материал;
- display name;
- lore;
- правила проверки предмета;
- расходование лицензии;
- создание по ПКМ.

### trade

Настройки торговли:

- timeout setup-сессий;
- чёрный список каталога BUY.

### movement

Настройки виртуального движения:

- включение;
- шаг обновления;
- периодическое сохранение в БД;
- скорость по умолчанию;
- радиусы проекции;
- ETA и критерий прибытия;
- поведение при атаке;
- auto-resume.

### projection

Настройки физической проекции:

- cooldown spawn/despawn;
- плавное следование;
- teleport correction;
- интервал синхронизации.

### interaction

Настройки антиспама взаимодействий, например:

- `cooldown-millis`

### route

Маршруты:

- включение;
- timeout настройки;
- мин/макс время остановки;
- максимум точек;
- фильтры по городам;
- loop mode;
- эффекты на остановке.

### notifications

Настройки очереди уведомлений для офлайн-владельцев.

### dynmap

Настройки интеграции с Dynmap:

- включение;
- interval update;
- marker-set-id;
- показ ETA.

### placeholderapi

Настройки интеграции с PlaceholderAPI:

- `enabled`
- `empty-value`
- `eta-format`
- `hp-decimals`

Если PlaceholderAPI отсутствует, этот раздел не ломает запуск плагина. Expansion просто не регистрируется.

### physical-caravan

Настройки физической проекции:

- включение;
- очистка orphaned entities;
- duplicate spawn protection;
- названия сущностей;
- поведение interaction;
- связь урона с системным HP.

### debug

Включает дополнительные диагностические логи и поведение, где это предусмотрено.

## HP

Главное правило:

```text
caravan.health — единственный реальный источник HP по умолчанию.
```

Это означает:

- именно HP каравана хранится как системное значение;
- сущности проекции не являются отдельным источником «настоящего» здоровья;
- визуальное здоровье мобов только отображает состояние каравана.

## config-version

Каждый из трёх файлов содержит:

```yml
config-version: 2
```

Этот ключ нужен для безопасного обновления ресурсов между версиями плагина.
