# Использование

## Как выдать лицензию каравана

Администратор выдаёт лицензию командой:

```text
/caravan admin givelicense <ник> [количество]
```

Пример:

```text
/caravan admin givelicense Steve 1
```

Если инвентарь игрока заполнен, лишние лицензии выпадут рядом с ним.

## Как игрок создаёт караван через ПКМ

1. Игрок получает предмет лицензии.
2. Берёт его в основную руку.
3. Кликает ПКМ по воздуху или блоку.
4. Плагин проверяет:
   - включены ли лицензии;
   - есть ли право `meltarion.caravans.use`;
   - не превышен ли лимит караванов;
   - является ли предмет правильной лицензией.
5. Если всё в порядке, создаётся караван с именем вида `Caravan-1`.

Важно:

- имена караванов по умолчанию и вручную идут **без пробелов**;
- используйте буквы, цифры, `_` и `-`.

## Как заспавнить физический караван

Игрок:

```text
/caravan spawn <identifier>
```

Самые удобные варианты идентификатора для игрока:

- номер из `/caravan list`;
- имя каравана;
- short ID;
- полный UUID.

Примеры:

```text
/caravan spawn 1
/caravan info Severnyy_karavan
/caravan delete 2
```

Условия:

- это должен быть его караван;
- игрок должен стоять внутри своего города Towny;
- Towny должен быть установлен;
- физические караваны должны быть включены.

## Как открыть setup GUI

Открытие GUI владельца/админа происходит:

- правым кликом по физическому каравану;
- или через админ-команду:

```text
/caravan admin setup <identifier>
```

В setup GUI доступны:

- склад;
- продажа;
- заявки на покупку;
- текущие сделки;
- информация;
- маршрут.

## Как загрузить предметы

Пока обычная удалённая команда открытия склада игроку **не добавлена специально**.

Для отладки и администрирования склад можно открыть так:

```text
/caravan admin open <identifier>
```

Предметы, лежащие в инвентаре каравана, используются:

- для SELL-операций;
- для хранения валюты;
- для приёма товара при BUY-операциях.

## Как создать SELL-операцию

На текущем этапе есть два практических варианта:

1. Через setup GUI владельца/админа:
   - открыть `Sell Items`;
   - выбрать стак;
   - ввести цену в чат.
2. Через debug-команду:

```text
/caravan admin sell <identifier> <slot> <price>
```

SELL-операция:

- резервирует слот;
- продаёт весь выбранный стак как одну сделку;
- блокирует свободное редактирование этого слота.

## Как создать BUY-операцию

Через setup GUI:

1. Открыть `Buy Orders`.
2. Выбрать категорию.
3. Выбрать материал.
4. Ввести через чат:
   - количество за сделку;
   - цену;
   - максимальный общий объём.

Или через debug-команду:

```text
/caravan admin buy <identifier> <material> <amount> <price> <max-total>
```

Для админов `<identifier>` может быть таким:

- полный UUID;
- short ID;
- `owner:index`, например `Anton:1`;
- `owner:name`, например `Anton:Severnyy_karavan`.

Если имя неоднозначно, используйте номер из `/caravan list` или short ID.

## Как работает публичная торговля

Посторонний игрок кликает по физическому каравану:

- если он владелец или админ, открывается setup GUI;
- если нет, плагин пытается открыть публичную торговлю.

Публичная торговля даёт:

- `Buy From Caravan` — покупка из SELL-операций;
- `Sell To Caravan` — продажа в BUY-операции;
- `Info` — информация о караване.

## Как работает ограничение Shop Plot

Публичная торговля доступна только когда:

- физическая проекция каравана стоит на `Towny Shop Plot`;
- караван находится в статусе `STOPPED`;
- торговая операция активна.

Если караван не на Shop Plot, посторонний игрок увидит сообщение об отказе.
